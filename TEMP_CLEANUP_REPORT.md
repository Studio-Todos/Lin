# Lin Codebase Cleanup Report

Comprehensive analysis of code quality issues across the Lin compiler codebase.
Generated by systematic exploration on test-stabilization branch.

---

## CRITICAL (will cause bugs or silent wrong output)

### 1. Parser.c: No operator precedence (`src/Parser.c:697-731`)
All binary operators parse at same level, left-to-right. `1 + 2 * 3` parses as `(1 + 2) * 3`.

### 2. Parser.c: First-error-only, no recovery (`src/Parser.c:215-216`)
A single parse error aborts the entire parse. Users get only one error per compilation.

### 3. Parser.c: `=` token broken (`src/Parser.c:177`)
`case '='` falls through the switch to `TOKEN_EOF`. Assignment `=` is not a valid token.

### 4. Parser.c: `%` and `,` -> `TOKEN_IDENTIFIER` (`src/Parser.c:186, 199`)
Modulo and comma both lex as identifiers. The parser cannot distinguish `rem` vs `%`.

### 5. Parser.c/Ast.c: Memory leak - `strdup`'d callee names never freed (`src/Parser.c:722`, `src/Ast.c:12-15`)
Every binary op like `1 + 2` allocates a `strdup("add")` that `freeAst` never frees.

### 6. Ast.c: `AST_FIELD_ACCESS.computed_index` not freed (`src/Ast.c:22-23`)
If `computed_index` is set, it leaks.

### 7. Lowering.c: Manual MLIR string payloads (`src/Lowering.c:205, 1566`)
```c
snprintf(payloadBuf, sizeof(payloadBuf),
    "  %%res = func.call @%s(%%env, %%arg, %%state) : (i64, i64, i64) -> i64\n", ...);
```
Embeds MLIR textual format inside string attributes parsed later. Should use MLIR C API builders.

### 8. Lowering.c: `realloc` return not checked (`src/Lowering.c:354`)
```c
fv->names = realloc(fv->names, sizeof(*fv->names) * fv->capacity);
```
If `realloc` returns NULL, original pointer is lost (leak + corruption).

### 9. Lowering.c: Unbound variable silently returns null value (`src/Lowering.c:675-677`)
Caller has no way to detect or handle this. Likely causes crashes downstream.

### 10. Lowering.c: `strcat`/`strncat` into `cleanNames[2048]` no bounds checks (`src/Lowering.c:715-738`)
Buffer overflow if MLIR op input spec is large.

### 11. Lowering.c: `typeNamesBuf[1024]` overflow (`src/Lowering.c:2209`)
Type declaration concatenation has no bounds check.

### 12. Lowering.c: Fixed 2-arg limit (`src/Lowering.c:2073, 2112`)
Loops use `i < 2` instead of actual arg count. Functions with >2 args silently ignore args 3+.

### 13. `parser_test.c` broken - tests reference dead AST types (`test/parser_test.c:167-200`)
`test_binary_op` expects `AST_BINARY`, but parser creates `AST_CALL`. `test_negative_number` expects `AST_UNARY`, but parser creates `AST_NUMBER` with negated value. `test_field_access` uses `.field`, but struct uses `.field_index`.

## HIGH (major refactoring debt)

### 14. main.cpp: Import resolution copy-pasted 3 times (`src/main.cpp:420-519, 1014-1092, 1154-1227`)
~300 lines of identical code for `build`, `check`, and `test` subcommands.

### 15. main.cpp: `runCompilationPipeline()` is 475 lines (`src/main.cpp:391-865`)
Should be split into ~5-7 functions.

### 16. main.cpp: `main()` is 376 lines (`src/main.cpp:869-1244`)
Subcommand implementations should be extracted.

### 17. Lowering.c: `lowerExpression()` is ~1455 lines (`src/Lowering.c:572-2027`)
The largest function in the codebase. Handles ~15 AST types inline.

### 18. Lowering.c: 6+ inline epsilon constructions ignoring `createEra` helper (`src/Lowering.c:248, 1529, 1540, 1613, 1623, 1763, 1779, 2127`)
The `createEra` function exists (line 549) but is not used at these sites.

### 19. Lowering.c: 15+ inline link constructions ignoring `linkValues` helper
Same pattern - the helper exists but isn't used everywhere.

### 20. Build system: Hardcoded Nix store path (`lib/dialect/CMakeLists.txt:1`)
```cmake
set(MLIR_TABLEGEN_EXE "/nix/store/mh0is1h5921i38ah1lkv2hgik69imp7f-mlir-18.1.8/bin/mlir-tblgen")
```

### 21. lit.cfg.py: Hardcoded `../build/` paths instead of CMake-configured (`test/lit.cfg.py:12, 16-18`)
Paths ignore `lit.site.cfg.py.in` which has proper `@CMAKE_SOURCE_DIR@`.

### 22. main.cpp: Hardcoded `gcc` linker, no env override (`src/main.cpp:780`)
System without `gcc` fails. Should check `CC`/`LIN_LD` env vars.

### 23. main.cpp: `LD_LIBRARY_PATH` parsed for `-L` flags (`src/main.cpp:786-804`)
Wrong env var. Should use `LIBRARY_PATH`.

### 24. main.cpp: `exit()` instead of `_exit()` after `fork()` (`src/main.cpp:838`)
`exit()` runs atexit handlers in child process.

### 25. flake.nix: Wrong `-DLLVM_DIR=$out` overrides env vars (`flake.nix:58-59, 115-116`)
`$out` is the Lin package output, not LLVM's.

### 26. flake.nix: `gtest` is a dead dependency (`flake.nix:43, 101, 161`)
No test uses Google Test.

### 27. main.cpp: `log()` with embedded `\n` produces double newline (`src/main.cpp:557`)

### 28. main.cpp: Memory leak on `realloc` failure in import resolution (`src/main.cpp:483-488`)

### 29. No CI pipeline - no `.github/workflows/` files
