# Lin Codebase Cleanup Report

Comprehensive analysis of code quality issues across the Lin compiler codebase.
Generated by systematic exploration on test-stabilization branch.

Updates marked with [DONE] = committed, [PARTIAL] = partially addressed.

---

## CRITICAL (will cause bugs or silent wrong output)

### 1. Parser.c: No operator precedence (`src/Parser.c:697-731`)
All binary operators parse at same level, left-to-right. `1 + 2 * 3` parses as `(1 + 2) * 3`.

### 2. Parser.c: First-error-only, no recovery (`src/Parser.c:215-216`)
A single parse error aborts the entire parse. Users get only one error per compilation.

### [DONE] 3. Parser.c: `=` token broken (`src/Parser.c:177`)
`case '='` now returns `TOKEN_ASSIGN` instead of falling through to `TOKEN_EOF`.

### [DONE] 4. Parser.c: `%` and `,` -> `TOKEN_IDENTIFIER` (`src/Parser.c:186, 199`)
`%` now returns `TOKEN_PERCENT`, `,` returns `TOKEN_COMMA`.

### 5. Parser.c/Ast.c: Memory leak - `strdup`'d callee names never freed
`strdup`'d callee names from binary ops leak. Needs careful fix (callee may be strdup, string literal, or source ptr).

### 6. Ast.c: `AST_FIELD_ACCESS.computed_index` not freed (`src/Ast.c:22-23`)

### 7. Lowering.c: Manual MLIR string payloads (`src/Lowering.c:205, 1566`)
Embeds `func.call @name(env, arg, state) : (i64, i64, i64) -> i64` as text. Should use C API builders.

### [DONE] 8. Lowering.c: `realloc` return not checked (`src/Lowering.c:354`)
Fixed in Phase 2: added `if (!tmp) { fprintf(stderr, ...); exit(1); }` guard.

### 9. Lowering.c: Unbound variable silently returns null value (`src/Lowering.c:675-677`)

### 10. Lowering.c: `strcat`/`strncat` into `cleanNames[2048]` no bounds checks (`src/Lowering.c:715-738`)

### 11. Lowering.c: `typeNamesBuf[1024]` overflow (`src/Lowering.c:2209`)

### 12. Lowering.c: Fixed 2-arg limit (`src/Lowering.c:2073, 2112`)
Loops use `i < 2` instead of actual arg count.

### [DONE] 13. `parser_test.c` broken - tests reference dead AST types (`test/parser_test.c:167-200`)
Fixed in Phase 2: assertions now match actual parser output (`AST_CALL`, `AST_NUMBER`, `.field_index`).
